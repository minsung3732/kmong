# 아이러브베트남 여행 예약 시스템 — 프로젝트 문서

> Google Apps Script + Google Sheets 기반 **서버리스** 여행사 예약 관리 시스템
> (별도 서버/호스팅 없이 Apps Script 웹앱 + 스프레드시트로 백엔드 구성)

---

## 1. 시스템 개요

| 항목 | 내용 |
|---|---|
| 백엔드 | Google Apps Script (`doGet` 라우팅 + `google.script.run` RPC) |
| DB | Google Sheets (스프레드시트 1개, 시트 여러 개) |
| 파일 저장 | Google Drive (여권 이미지 등, **공유 드라이브 가능**) |
| 프론트 | HTML Service (Pretendard 폰트, 연한 하늘색 톤, 40~50대 친화적 대형 UI) |
| OCR | Google Drive 내장 OCR (이미지 → 구글 문서 변환), MRZ 파싱 |
| 인증/배포 | 웹앱 배포, **"다음 사용자로 실행: 나" / "액세스: 모든 사용자"** |

### 핵심 설계 원칙
- **예약번호는 사전 발급** — 자동 생성 안 함. 고객이 안내받은 번호를 직접 입력.
- 모든 조회 기준: **대표자 영문 성함 + 예약번호** 조합.
- 여권 정보는 예약과 **분리** — 예약 확정 후 별도 페이지에서 등록.
- 상품 목록은 `상품 구성` 시트에서 **동적 로드** (코드 수정 없이 상품 추가 가능).
- Sheets에서 읽은 **Date 객체는 반드시 문자열로 변환** 후 클라이언트에 반환 (직렬화 null 방지).

## 3. 파일 구성

### Apps Script (.gs)
| 파일 | 역할 |
|---|---|
| **Code.gs** | 메인 백엔드 — 라우팅, 신청서/예약변경/차량렌트/여권 CRUD, 마이그레이션 |
| **CardGen.gs** | **고객 예약 카드 html** — `고객 정보 입력` 시트 기반 카드 생성 + Drive 저장 + 링크 |
| **ResConfirm.gs** | **예약 확인 카드 html** — `예약 정보` 시트(담당자 수기) 기반 확인서 생성 + 링크 |

> `CARD_SS_ID`, `WEBAPP_BASE_URL` 등 전역 상수는 **단 한 파일에서만 선언** (중복 선언 시 `Identifier already declared` 오류). `ResConfirm.gs`는 `CardGen.gs`의 `WEBAPP_BASE_URL`을 공유.

### HTML (HTML Service)
| 파일 | 역할 | URL |
|---|---|---|
| **신청서.html** | 고객 신청서 (3탭) | `...exec` |
| **여권등록.html** | 여권 정보 등록 (탑승자별) | `...exec?page=passport` |

### 웹앱 라우팅 (`doGet`)
```
...exec                          → 신청서.html (메인)
...exec?page=passport            → 여권등록.html
...exec?page=card&bookingNo=XXX  → 고객 예약 카드 (CardGen)
...exec?page=confirm&order=XXX   → 예약 확인 카드 (ResConfirm)
```

## 4. 시트 구조

### 4-1. `고객 정보 입력` (고객 신청서 저장)
| 열 | 헤더 | 비고 |
|---|---|---|
| A | 접수시간 | 자동 (KST) |
| B | 예약번호 | 사전 발급, 고객 입력 |
| C | 상품명 | |
| D | 상품유형 | 단독 투어 / 1박 포함 등 |
| E | 투어날짜 | |
| F | 대표자영문성함 | 조회 키 |
| G | 픽업호텔 | |
| H | 드랍호텔 | |
| I | 출발시간 | 단독 투어 시 |
| J | 성인 | **여권 등록 인원수 기준** |
| K | 소아 | **여권 등록 인원수 기준** |
| L | 소아만나이 | |
| M | 옵션메모 | |
| N | 베드타입 | 1박 포함 상품만 |
| O | **여권 정보** | 셀 값=여권번호들(쉼표), **셀 메모=탑승자별 상세** |
| P | 비고 | |
| Q | **컨펌번호** | 담당자 수기 입력 → 카드에 표시 |
| R | **(카드 링크)** | CardGen 자동 입력 (`COL_CARD_LINK=17`) |

### 4-2. `여권 정보` (여권 raw data 정리)
여권 저장 시 자동 적재 (예약번호 기준 **upsert**).
```
접수시간 / 예약번호 / 상품명 / 상품유형 / 투어날짜 / 대표자영문성함 / 여권 정보
```
- `여권 정보` 열: 셀 값=여권번호들, **셀 메모=탑승자별 상세 블록**

### 4-3. `예약 정보` (담당자 수기 입력 → 예약 확인 카드)
**헤더가 3행에 위치해도 자동 탐지** (`getResInfoMap_`가 1~15행 검색).
```
일자 / 예약번호(주문번호) / 유입 플랫폼 / 상품명 / 투어 날짜 /
대표자명 / 인원 정보 및 옵션 / 연락처 / 판매금액 / HTML 링크 / 알림톡 발송여부
```
- 행 입력 시 `onEditResInfo` 트리거가 `HTML 링크`에 확인서 URL 자동 생성

### 4-4. `예약 변경 요청` (이력 누적)
```
접수시간 / 예약번호 / 상품명 / 상품유형 / 투어날짜 / 대표자영문성함 /
픽업호텔 / 드랍호텔 / 출발시간 / 성인 / 소아 / 소아만나이 / 옵션메모 / 베드타입 / 변경메모 / 처리상태
```

### 4-5. `차량 렌트 신청`
```
접수시간 / 예약번호 / 운행날짜 / 대표자영문성함 / 픽업호텔 / 드랍호텔 / 출발시간 / 운행일정 / 비고 / 최종수정시간
```
- 신규는 append, 변경은 **기존 행 in-place 수정**

### 4-6. `상품 구성` (수동 관리)
```
A: 상품명 / B: 상품유형 / C: 출발시간(쉼표 구분, 예 "07:00,08:00")
```

## 5. 신청서.html (메인, 3탭)

### 탭 1 — 고객 정보 입력
- 상품 드롭다운 (`getProductList()` 동적 로드)
- 상품유형 `단독 투어` → 출발시간 드롭다운 노출
- 상품유형에 `1박` 포함 → 베드타입 노출
- 사전 발급 **예약번호 직접 입력**
- 인원 카운터(성인/소아), 소아>0 시 만나이 입력
- 제출 → `고객 정보 입력` append, **성공 시 상단 커튼 토스트**

### 탭 2 — 예약 변경
- 영문성함+예약번호 조회 → 내역 표시 → 수정 폼
- **필수 미입력 시 해당 필드에 인라인 에러 + 스크롤** (상단 점프 X)
- 제출 버튼 **"제출 중…" 로딩** 표시
- 제출 → `예약 변경 요청` append

### 탭 3 — 차량 렌트
- 서브탭: 신규 신청 / 예약 변경
- 신규 → `차량 렌트 신청` append
- 변경 → 조회 후 기존 행 in-place 수정

### 공통 UI 패턴
- **성공 알림**: 상단에서 내려오는 커튼형 토스트 (자동 사라짐)
- **에러 알림**: 입력 폼 근처 인라인 표시
- 조회 버튼: 로딩 스피너 + **15초 타임아웃** 처리

## 6. 여권등록.html (탑승자별 여권 등록)

### 흐름
```
Step1 조회(영문성함+예약번호) → Step2 탑승자별 입력 → Step3 완료
```

### Step2 — 탑승자 수만큼 폼 자동 생성
- 조회 시 `성인(J) + 소아(K)` 인원수만큼 **탑승자 카드 동적 생성**
- 각 탑승자: **직접 입력 / 이미지 업로드** 토글 (입력 필드는 **항상 표시**)
- 입력 항목: 영문성명 / 성별 / 국적(약 170개국) / 생년월일 / 여권만료일 / 여권번호
- **전원 입력 후 한 번에 저장** → `savePassportInfo({_rowIndex, passengers:[...]})`

### 이미지 업로드 + OCR
- **HEIC(아이폰) → JPG 자동 변환** (`heic2any` CDN, 클라이언트 변환)
  - 최신 HEIC(HEVC) 변환 실패 시 → 원본 업로드 + 직접 입력 안내
  - ⚠️ 모바일 Safari는 사진 선택 시 대개 JPG로 전달되어 정상 작동
- 업로드 → Drive 저장(`Drive.Files.create`, **공유 드라이브 지원**) → OCR
- **MRZ 파싱** (TD3 여권 기계판독 2줄):
  - L1(이름 줄): `P`로 시작 & `<<` 포함. **0-1 종류 / 2-4 발급국 / 5- 이름** (한국 여권 종류 `PM` 2글자 대응)
  - L2(데이터 줄): **숫자 가장 많은 줄** 선택 (`<` 없을 수 있음, 인쇄 텍스트 자동 제외)
    - 0-8 여권번호 / 10-12 국적 / 13-18 생년월일 / 20 성별 / 21-26 만료일
  - 국적코드(KOR 등) → 국적명 자동 선택, 성별(M/F) 자동 선택

### 저장 결과 (2곳 동시 기록)
1. `고객 정보 입력` 시트 해당 행 **`여권 정보` 셀 + 메모**
2. `여권 정보` 시트 (예약번호 upsert) **셀 + 메모**

메모 형식:
```
[탑승자 1]
KIM MINSUNG
남
대한민국
1994-05-27
M66673992
2029-01-25
이미지: https://drive.google.com/file/d/.../view

[탑승자 2]
...
```

## 7. 카드 생성 (2종)

### 7-1. 고객 예약 카드 html (CardGen.gs)
- 데이터: `고객 정보 입력` 시트
- 트리거: `onEditIssueCard` (A~Q열 편집 시 해당 행 재발행) / `issueAllCards()` 일괄
- 카드 링크 → **R열** (`COL_CARD_LINK=17`, `DATA_COL_END=16`)
- 표시: 상품정보 / 예약정보 / 인원 / 옵션 / **컨펌번호(있으면 강조)** / **여권 정보(토글 펼치기·숨기기)** / 비고
- 상단: `- 예약번호 -` 라벨 + 큰 번호
- **여권 정보 토글**: 기본 숨김 → "여권 정보 펼치기 ▼" 클릭 시 표시, "숨기기 ▲"로 재접기

### 7-2. 예약 확인 카드 html (ResConfirm.gs)
- 데이터: `예약 정보` 시트 (담당자 수기 입력)
- 트리거: `onEditResInfo` (행 입력 시 `HTML 링크` 자동 생성) / `issueAllResInfo()` 일괄
- URL: `...exec?page=confirm&order=주문번호`
- 표시: 상품정보 / 예약자정보(대표자명·연락처·일자) / 인원·옵션 / **판매금액(원 단위 콤마)**
- 디자인: 고객 예약 카드와 동일 톤

## 10. 트러블슈팅 (이미 해결된 이슈)

| 증상 | 원인 | 해결 |
|---|---|---|
| 조회 무한 로딩 / null 반환 | Sheets Date 객체 직렬화 실패 | 서버에서 모든 값 String 변환 후 반환 |
| 조회해도 화면 안 뜸 | 구버전 웹앱 배포 | 새 버전 재배포 |
| 이미지 업로드 "액세스가 거부됨: DriveApp" | 공유 드라이브 미지원 | `DriveApp` → 고급 Drive 서비스(`Drive.Files.*`, `supportsAllDrives:true`) |
| HEIC 변환 실패 | 최신 HEVC 코덱 미지원 | 원본 업로드 폴백 + 직접 입력 안내 (모바일은 대개 JPG 전달) |
| OCR 인식값 오류(이름에 R, 필드 빈값) | MRZ 줄/필드 위치 오판 | `P` 시작 줄 탐지 + 5번째부터 이름 + 숫자 최다 줄을 데이터 줄로 |
| `Identifier 'CARD_SS_ID' already declared` | 전역 상수 중복 선언 | 상수는 한 파일에서만 선언 |
| 함수 실행 무한 로딩 | 독립 스크립트에서 `getUi().alert()` 사용 불가 | `Logger.log()`로 대체 |
| 모바일에서 PC 레이아웃 | iframe 내부 viewport 무시 | `doGet`에서 `.addMetaTag('viewport', ...)` 추가 |

### 🔖 저장 규칙 (중요)
> 사용자가 **"깃허브에 저장해줘"(업로드해줘)** 라고 하면:
> 1. **버전 폴더**를 새로 만들어 현재 모든 파일을 **스냅샷 복사**한다.
>    - 폴더명 형식: `V<버전> (YY.MM.DD HH:MM)` — 예: `V1.0 (26.06.18 15:36)`
>    - 버전은 직전 최고 버전에서 **0.1씩 증가** (V1.0 → V1.1 → V1.2 …)
>    - 시각은 **업로드 시점의 실제 시각**(KST)으로 기입
> 2. **인수인계서(PROJECT.md 등 문서 포함) 전체**를 함께 저장한다. (코드만 올리지 말 것)
> 3. 루트의 작업 파일은 그대로 두고(라이브 편집용), 버전 폴더는 **스냅샷 이력**으로 누적한다.

스냅샷 대상 파일: `Code.gs`, `CardGen.gs`, `ResConfirm.gs`, `AlimTalk.gs`, `체크박스_B열복구.gs`, `신청서.html`, `여권등록.html`, `PROJECT.md`, `manual.md`

```bash
# 다음 버전 폴더 만들고 스냅샷 → 커밋 → 푸시
STAMP=$(date "+%y.%m.%d %H:%M")
VDIR="V<다음버전> ($STAMP)"
mkdir -p "$VDIR" && cp <대상파일들> "$VDIR"/
git add -A && git commit -m "<변경 요약> ($VDIR)" && git push origin main
```
